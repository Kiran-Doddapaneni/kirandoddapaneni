2. Vision Statement
To empower SOC analysts with an AI-driven security platform that unifies Wazuh SIEM and OpenCTI threat intelligence, enabling natural language querying of security alerts, real-time IoC enrichment, and automated threat response — reducing investigation time and bridging the gap between raw security data and actionable intelligence.
________________________________________
3. Architecture Design (Docker Compose)
System Layers
●	Data Sources: Linux/Windows endpoints, Network devices, Cloud and containers
●	Collection: Wazuh Agent deployed on each monitored endpoint
●	SIEM Core: Wazuh Manager (detection and analysis), Wazuh Indexer (alert storage and search), Wazuh Dashboard (visualisation and investigation)
●	CTI Layer: OpenCTI — bidirectional IoC enrichment with Wazuh
●	Active Response: Automated remediation triggered by high-confidence alerts
●	AI Layer: AI Chatbot Backend — LLM, RAG, Wazuh API, OpenCTI API
●	Analyst Interface: SOC Analyst chat UI
Docker Compose Services
●	wazuh-manager
●	wazuh-indexer (OpenSearch — port 9200)
●	wazuh-dashboard (port 443)
●	opencti (port 8080)
●	opencti-worker
●	rabbitmq
●	minio (S3 object storage)
●	ai-chatbot-api (FastAPI — port 8000)
●	ai-chatbot-ui (React — port 3000)

5. Scenarios
Scenario 1 — Detecting a Suspicious Login Attack Written by: Bharadwaz Cirasanagandla
Objective: Detect and respond to a suspicious login attempt quickly. Actor: Alex Muller (SOC Analyst)
Activities:
●	Alex is monitoring the SIEM dashboard during his shift when a login alert is triggered from an unusual IP address.
●	The SIEM Agent automatically enriches the alert using OpenCTI and shows that the IP is linked to previous brute-force attacks.
●	The system visually maps the IP with related attack patterns and highlights it as high risk.
●	Alex reviews the alert details and confirms it is suspicious.
●	The bot suggests actions such as blocking the IP and creating an incident ticket.
Problem / Gap: Without enrichment, Alex would need to manually check multiple sources, wasting time and possibly missing critical threats.
Resolution: The SIEM Agent integrates OpenCTI and uses the bot to automatically enrich and prioritize alerts, allowing Alex to respond instantly and efficiently.

6. User Stories
1.	As a SOC Analyst, I want to ask the chatbot about recent alerts in plain English so that I can triage faster without writing OpenSearch queries.
2.	As a SOC Analyst, I want the chatbot to enrich an IP or file hash with threat intelligence so that I know if it is a known IoC.
3.	As a SOC Analyst, I want the system to suggest automated remediation actions so that I can respond to threats quickly.
4.	As a SOC Analyst, I want the chatbot to map alerts to MITRE ATT&CK techniques so that I can understand the attack pattern.
5.	As a Security Engineer, I want to configure the bot to automatically respond to repeated threat patterns so that manual effort is reduced.
6.	As a Security Engineer, I want to see all automated actions logged for audit purposes so that I can review and verify what the system did.
7.	As a Cybersecurity Student, I want to explore simulated attack scenarios with visual graphs so that I can understand how threats are detected.
8.	As a Cybersecurity Student, I want to ask the bot why an alert is critical so that I can learn the reasoning behind threat detection.

7. Features
Wazuh SIEM Core
●	Real-time log collection from Linux, Windows, cloud, and containers
●	Rule-based alert detection and correlation
●	File Integrity Monitoring (FIM)
●	Vulnerability scanning
●	Active Response — automated IP blocking, process kill, quarantine
OpenCTI Integration
●	Bidirectional IoC enrichment — IP, hash, and domain lookups
●	MITRE ATT&CK technique mapping
●	Threat actor and campaign context
●	STIX-format intelligence sharing
AI Chatbot
●	Natural language querying of Wazuh alerts
●	RAG-powered responses grounded in real alert data
●	IoC lookup through OpenCTI API
●	Alert summarisation and triage assistance
●	MITRE ATT&CK technique explanation
●	Audit log of all automated actions
●	Visual graph of entity relationships (IP, user, system)
